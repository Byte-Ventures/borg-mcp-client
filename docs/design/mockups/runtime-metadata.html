Implementation acceptance

  • Use the exact labels Agent CLI, Reported model, Working repo, and subordinate Origin.
  • Render the advisory note once per identity region, before its seats. Do not repeat it for every seat.
  • Compact surfaces show canonical public owner/repo. Only an explicit detail surface shows sanitized canonical origin.
  • Escape control, ANSI, carriage-return, line-separator, bidi-control, and Markdown-breaking characters before rendering any metadata. Never render raw unknown enum values.
  • Never show a local path, userinfo, credential, query, fragment, token-like text, or raw rejected value. Hostile input must not become a link target.
  • Status is always conveyed in words, not color or icon alone. Label and value remain adjacent in reading order when Markdown and styling are removed.
  • Any visual disclosure uses native semantics or an explicitly named button, is keyboard operable, has visible focus, and retains expanded state. Full values are not tooltip-only.
  • Known metadata never changes role, authority, grants, liveness, wake behavior, activity, last_seen, or routing. Unknown and failure states do not imply degraded health.
  • Tests cover Claude Code, Codex, OpenCode, known and unknown model, unreported older seat, temporary failure with and without a previous value, rejected hostile origin with and without a previous value, maximum lengths, and 80/120-column snapshots for all three surfaces.
