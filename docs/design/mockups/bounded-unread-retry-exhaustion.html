The MCP result is an error result. The command is presented as copyable inline text, not as an automatically executed action.

Behavior contract

  • Ordinary burst reads and any retry that later succeeds show no retry, waiting, or recovery text.
  • The saved unread position advances only after the complete page is ready to show. Exhaustion leaves the position at the last successfully shown entry, so the next command safely resumes the unread drain.
  • This terminal result contains no transport status, protocol term, socket detail, URL, path, host, credential detail, attempt count, stack, or internal error text.
  • The result names the failed operation, states cursor safety, and gives one immediate recovery action. It does not imply that activity was lost or that the user must re-assimilate.

Implementation boundary

Apply this state only to retry-eligible unread-log failures after the bounded retry policy is exhausted. Validation, authorization, and other non-transient failures retain their own specific recovery states.
