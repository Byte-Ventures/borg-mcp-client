Private Borg state unavailable

Terminal CLI result when Borg cannot safely use its private local state.

Terminal result

Borg could not safely prepare its private local state.

No Borg server or cube change was made.

Before retrying, verify that Borg's private-state directory and parent directories are real, owned by your account, and not accessible to other users. Verify the same for its Borg files, then run the same command again.

The command exits with an error. The recovery action is shown as copyable inline text and is not executed automatically.

Behavior contract

  • A missing private root, or a current-user-owned root that is not writable by group or other users, is repaired to private mode without this message.
  • Show this state before any local state read, credential persistence, authority discovery, cube creation, attach request, or other remote mutation.
  • Do not expose a path, filesystem implementation detail, race condition, stack, credential, token, or state-file contents.
  • Do not suggest deleting files, changing permissions recursively, re-assimilating, or trying a different role. The same command is the recovery path after the user corrects the directory.
